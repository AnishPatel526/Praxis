# Incident Summary

**Incident ID:** INC-4821

On [Date of Incident, if known, or "recently"], users experienced intermittent failures during payment validation, leading to failed transactions and degraded user experience. Symptoms included `stripe.customers.retrieve` API call failures, resulting in the inability to process payments.

# Root Cause Analysis

The root cause was identified as insufficient resilience in the `validatePaymentToken` middleware against transient external API failures. Specifically, the `stripe.customers.retrieve` API call lacked a retry mechanism. Intermittent network issues, Stripe API rate limiting, or temporary service unavailability could cause the initial API call to fail, immediately rejecting the payment token without any attempt to recover, leading to user-facing errors.

# Resolution

A `retryWithBackoff` utility function was implemented (`utils/retry.ts`) to introduce exponential backoff and retry logic for unreliable operations. This utility was then integrated into the `validatePaymentToken` function (`auth/middleware.ts`) to wrap the `stripe.customers.retrieve` API call. This change ensures that transient failures are automatically retried up to a configured maximum, significantly improving the robustness of the payment validation process. The fix was deployed via Merge Request: `https://gitlab.com/AnishPatel526/praxis-test/-/merge_requests/5`.
